如何用hugo搭建一个博客

Written by tangallison2 with ♥ on April 19, 2020 in

在已有github账户和已安装Cmder和VSCODE的情况下，使用hugo搭建一个博客 当然如果没有注册github账户和未安装Cmder和VSCODE软件，可以先去安装啊

  1. 首先安装hugo
  • 点它hugo安装包链接
  • 选择适合你的版本号安装，例如windows用户可选hugo_XXXXX_Windows-64bit.zip
  • 安装后，可在Cmder软件中输入
hugo version

如果显示出版本信息，则安装成功。

  1. Google里面搜索hugo,进入hugo官网
  • 点击quickstart,png
  • 先在Cmder进入你放文件的目录，从Step 2开始一步一步更着官网的代码在Cmder上输入,下面的XXXX你可以替换成你的github用户名。

step2

hugo new site XXXX.github.io-creator

在之后建立的github远程仓库命名必须为：XXXX.github.io,不用深究，后面会说到。

step3 建立XXXX.github.io-creator目录的本地仓库，并下载一个博客主题

cd XXXX.github.io-creator
git init
git submodule add https://github.com/budparr/gohugo-theme-ananke.git themes/ananke
echo 'theme = "ananke"' >> config.toml

step4

hugo new posts/my-first-post.md

这步之后生成一个my-first-post.md的博客，用VSCODE打开后修改内容

draft:false

可以在博客里面写一些内容

step5

hugo server -D

效果是启动hugo服务器，ctrl+点击http://localhost:1313/，就能查看my-first-post.md内容。

step6 用VSCODE打开XXXX.github.io-creator，找到并打开文件config.toml，用更私人的标题替代下面的对应代码。

baseURL = "http://tangjixuan.top/"
languageCode = "en-us"
title = "My New Hugo Site"
theme = "ananke"

step7 在Cmder中输入

hugo -D

这一步会生成一个public目录

  1. 在XXXX.github.io-creator目录下新建.gitignore文件，编辑内容为/public/,意思是当前仓库不要储存public目录。

  2. 在public目录下建立本地仓库

git init
git add .
git commit -v

并输入提交信息

  1. 在github上建立一个名为XXXX.github.io的远程仓库

  2. 把public本地仓库上传到远程仓库

git remote add origin git@github.com:XXXX/XXXX.github.io.git
git push -u origin master
  1. 上传成功后，进入对应的github远程仓库，点击settings,找到Github Pages里source下面的按钮改为 master branch ,点击Github Pages里的 https://XXXX.github.io/,就能看到博客了。
  2. 可以在namesilo或者阿里云上面注册一个域名，把Github Pages的Custom domain替换成自己注册的域名，同时也可点Custom domain后面的learn more 采集IP地址，在购买域名的网站上配置域名的DNS，可在Cmder里输入
nslookup XXXX

XXXX需替换成自己购买的域名，测试是否配置成功。若测试成功，域名即可显示博客。
